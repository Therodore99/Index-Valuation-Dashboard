# 指数区间位置展示项目（V2 - iFinDPy版）

## 一、项目目标

用 Python 编写一个可复用、可扩展、可维护的指数数据分析项目。

当前阶段目标是：
以恒生科技指数为示例，展示多个指标在指定时间范围内的历史区间位置。

当前支持的指标包括：
- 指数收盘价（close）
- 市盈率（pe）
- 市净率（pb）

展示方式参考目标样式：
- 一条横向区间线表示历史区间
- 左侧标注区间最低值
- 右侧标注区间最高值
- 中间用红点标注当前值
- 显示当前值在区间中的相对位置

本项目后续会扩展到更多指数，因此代码必须具备良好的可复用性。
新增其他指数时，应尽量只修改配置，而不需要重写主流程代码。

---

## 二、数据来源

- 使用 iFinDPy 获取数据
- 不使用 iFinD HTTP API
- 本地环境已具备 iFinDPy 使用条件

## 四、时间范围

- 当前默认支持近1年区间展示
- 时间范围计算方式：从当前日期往前365天
- 若交易日不足365天，则以实际返回交易日为准
- 当前值定义为：距离当前日期最近一个交易日的数值

代码设计上应为后续扩展到近5年保留能力。

## 五、数据缓存与复用

为了减少 iFinD 数据消耗，必须实现本地 CSV 缓存机制。

## 1. 缓存规则
每个指数、每个指标分别保存一个 CSV 文件。

推荐目录结构：

data/
  hstech/
    close.csv
    pe.csv
    pb.csv

## 2. CSV要求
每个 CSV 至少包含两列：
- date
- value

## 3. 增量更新逻辑
脚本运行时应：

1. 先检查本地是否已有对应 CSV
2. 如果已有：
   - 读取已有历史数据
   - 判断最新日期是否已覆盖到最近交易日
   - 若未覆盖，则仅从 iFinD 获取缺失日期的数据
   - 将新旧数据合并、去重、按日期排序后重新保存
3. 如果没有：
   - 从 iFinD 拉取完整区间数据并保存 CSV

## 4. 目标
例如当需要近5年数据时：
- 不重复获取历史已有数据
- 只补充最新缺失部分
- 降低 iFinD 调用消耗

## 六、数据逻辑

对于每个指数、每个指标，需要获取一段时间内的时间序列数据，并计算：

- 区间最低值 min_value
- 区间最高值 max_value
- 当前值 current_value（最近交易日）
- 区间位置比例 position_pct

计算公式：

position_pct = (current_value - min_value) / (max_value - min_value) * 100

当 max_value == min_value 时，需要做保护处理，避免除零错误。

### 8.7 指标类型差异与分位使用规范

#### 一、指数值（close）

指数价格属于非平稳时间序列，长期具有趋势性，因此：

- 不具备稳定历史中枢
- 不适合直接生成“低估 / 高估”类估值判断
- 不应使用历史分位生成估值结论

close 允许表达：

- 当前位于近1年区间中部
- 接近阶段高点 / 低点
- 位于区间上沿 / 下沿附近

close 禁止表达：

- 低估 / 高估
- 偏高 / 偏低估值
- 任何估值判断类结论

---

#### 二、估值指标（PE / PB）

PE、PB 属于相对估值指标，具有明确经济含义，因此：

- 必须计算历史分位
- 必须使用统一估值标签体系
- 必须生成基于分位的结论

当前点标注建议格式：

- 当前 21.4
- 40%分位
- 中性

或在图内简化为：

- 当前 21.4
